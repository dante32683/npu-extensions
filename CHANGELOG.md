- **All extensions — preferences UX:** Moved single-command preferences from the extension-level "Preferences" panel into per-command sections (Raycast's command-level `preferences` array), so the Extensions settings UI now groups them under the command they affect. **No behavior change** — defaults and runtime semantics are identical. Moves: `npu-awake-ext` (`defaultAwakeMode` → `awake`, `defaultDuration` → `awake-for`, `defaultUntilTime` → `awake-until`, `defaultScheduleStart/End/Days` → `awake-natural`). `npu-notes-ext` (`prefillFromClipboard` → `add-note`, `semanticSearchDebounce` / `maxSemanticChecks` / `maxSemanticResults` → `search-notes`). `npu-image-editor-ext` (`defaultJpegQuality` / `clipboardOutputDir` → `modify-image`). `npu-dev-toolbox-ext` (`commitStyle` → `commit-message`). `npu-organize-ext` (`skipOnBattery` / `debounceMs` / `ignorePattern` → `start-screenshot-watcher`; the Status view's _Apply Current Preferences to Running Watcher_ action was removed since `getPreferenceValues` from Status's context cannot read Start's scoped prefs — re-run **Start Screenshot Watcher** to apply changes, which always rewrites `config.json` and the keeper hot-reloads on mtime change; the Status view's _Start_/_Stop_ buttons now delegate via `launchCommand` so the launched command sees its own scoped prefs). `npu-text-tools-ext` was left unchanged — every preference is read by ≥2 commands. Shared prefs that stay at extension level: `showSuccessToasts`, `ensureModelReady`, `showLidNote`, `defaultScaleFactor`, `autoOpenResult`, all dev-toolbox terminal/IDE settings, `foregroundDetectionTimeoutMs`, `notesFolder`, organize watch/naming/extensions/size/slug-tokens, and the text-tools selection prefs.

### Added

- **NPU Organize (`npu-organize-ext`) — Phase 2 (`docs/FORWARD_ROADMAP.md` §6 phase 2):** Resident watcher with battery skip, debounce, and shared state cursor. New `OrganizeKeeper.exe` (BCL-only, self-contained .NET 8 publish to `assets/bin/`) runs detached and monitors the configured folder via `FileSystemWatcher`. Per-path **debouncing** (`debounceMs` pref, default 1500ms) coalesces noisy `Created/Changed/Renamed` bursts; the keeper additionally polls the file for exclusive-read access for up to 5s to avoid racing Snipping Tool / Game Bar writes. **Battery skip** via `GetSystemPowerStatus` P/Invoke when `skipOnBattery` pref is on (manual Raycast commands always run regardless of power). New optional **ignore pattern** regex pref. Three new Raycast commands: `Start Screenshot Watcher` (no-view), `Stop Screenshot Watcher` (no-view; writes `stop.flag` then falls back to `process.kill(pid)` after 3s), `Screenshot Watcher Status` (view; running/stopped, PID, counters, last activity, recent log tail, _Apply Current Preferences to Running Watcher_ action). State lives outside Raycast's support folder at `%LocalAppData%\NpuOrganize\` (`config.json`, `state.json`, `stop.flag`, `organize.log`); the keeper **hot-reloads** when `config.json` mtime changes. Manual `Rename New Screenshots` now updates the same `state.json` cursor so the keeper status view shows a unified counter. **Slug parity** between the C# keeper and the TS Raycast command is enforced by `OrganizeKeeper.exe parity-check` (BCL-only inline assertions mirroring the 28 Vitest cases, including hardcoded FNV-1a reference values).
- **NPU Organize (`npu-organize-ext`)** — phases 0–1 (initial release). Two Raycast commands (`Rename New Screenshots`, `Dry Run Screenshot Rename`) scan the configured Screenshots folder (default `%UserProfile%\Pictures\Screenshots`), describe each image on-device, and rename it to `{YYYY-MM-DD}_{slug}.{ext}` (matches the `npu-notes-ext` convention) or `{slug}.{ext}`. New sparse-package bridge `NpuOrganizeBridge.Identity` (`assets/bin/NpuBridge.exe`) with verb `screenshot-title <imagePath> [--ensure-ready] [--no-ocr]` calling `Microsoft.Windows.AI.Imaging.ImageDescriptionGenerator.DescribeAsync` (BriefDescription / "caption" scenario, same API as the AI Dev Gallery "Describe Image WCR" sample) plus an optional `Windows.Media.Ocr.OcrEngine` grounding excerpt; emits one JSON line `{ status, description, confidence, ocrExcerpt, elapsedMs }`. Deterministic TS slug pipeline (lowercase, `[a-z0-9-]`, diacritic strip, stopword filter, configurable token cap, 60-char length cap) with `screenshot-{8-char FNV-1a hash}` fallback when the description is empty or low-confidence — never writes raw model text on disk. Collision resolver appends `-2`, `-3`, ...; "Skip Already-Named Files" prevents rename loops. Vitest suite (28 tests) covers slug + filename + collision logic offline. Bridge uses `Microsoft.WindowsAppSDK 2.0.1` + `Microsoft.WindowsAppSDK.AI 2.0.185` (same layout as `npu-image-editor-ext`). `register-bridge.ps1` and `EXTENSION_REGISTRY.md` updated.
- **NPU Text Tools (`npu-text-tools-ext`):** **Paste Selection** (`no-view`) and **Review Selection** (`view`) commands per rewrite mode; Raycast global hotkeys target these for in-place rewrite. `TextSelectionHelper.exe` (`send-copy` / `send-paste`) plus sentinel-based clipboard capture; paste uses `Clipboard.paste` after `closeMainWindow`. New prefs: **Quick custom instruction**, **Selection copy delay**. Shared `phi-rewrite-bridge` module; see `selection-helper/README.md` to publish the helper.
- **Suite (roadmap §1)** — Cohesive Raycast **preferences** across all five extensions: success toasts, optional **Ensure AI model readiness** (`--ensure-ready` on Phi/imaging bridges), notes semantic search tuning (debounce, max checks/hits), image editor auto-open + OCR text open, awake default duration/until time + Smart Awake schedule fallbacks, dev toolbox **workspace detection timeout** for PowerShell foreground probes. Bridge headers document readiness + link `docs/REWRITE_INFO.md`.
- **NPU Image Editor**: `Remove Background` command now runs the real NPU pipeline (same as Modify Image) instead of a placeholder scaffold.
- **NPU Awake (`npu-awake-ext`)**: Keyboard-first sleep prevention. Supports indefinite toggle, timed duration, "awake until" specific clock time, and screen-off mode. **Note: Currently being upgraded to "Smart Awake" with NPU natural language parsing.**
- **Smart Note Taker (`npu-notes-ext`)**: Phi-Silica backed note formatting and classification. Includes `add-note` for sloppy note filing and `browse-notes` for categorized listing/searching.
- **Notes discovery (`npu-notes-ext`)**: Added `find-related` and `search-notes`. Search does live keyword matches and a debounced Phi-Silica semantic fallback with caching + early-stop.
- **Sticker Maker (`npu-image-editor-ext`)**: Creates a 480×480 transparent WebP sticker from a selected image using on-device foreground extraction with smart auto-crop and a safe center-crop fallback.
- **NPU Dev Toolbox (`npu-dev-toolbox-ext`)**: New extension with three commands. **Open Workspace** uses the “last interacted” File Explorer folder (persisted as `last-explorer-folder`), lists subfolders as workspaces, and opens a chosen folder in a configurable terminal (Windows Terminal / pwsh / powershell / cmd / custom `.exe` or `.lnk`) and a configurable IDE (Cursor, VS Code, Windsurf, JetBrains IDEs, Sublime Text, Notepad++, custom `.exe` or `.lnk`) plus Explorer—any combination. **Workspace History** stores up to 20 previously opened folders (remove/clear supported) and exposes the same open actions. **Commit Message** detects the active git repo, gathers diff/log/branch context, and asks Phi-Silica for a JSON `{subject, body}` (Conventional Commits or Plain). Adds new sparse-package bridge `NpuDevToolboxBridge.Identity` registered via `register-bridge.ps1`.
- Documentation hub: `CONTRIBUTING.md`, `docs/RUNBOOK.md`, per-extension `NOTES.md`, and root stub files `AGENTS.md` / `CLAUDE.md` / `GEMINI.md` pointing at the same workflow.

### Fixed

- **NPU Dev Toolbox (`npu-dev-toolbox-ext`):** **Open in IDE** no longer leaves a visible **CMD** window beside the editor: the `cmd /c start …` helper used for `.cmd` / PATH launchers now runs with **`detached: false`** so **`windowsHide` actually hides the console** (Node quirk: `windowsHide` + `detached: true` is unreliable on Windows — [nodejs/node#21825](https://github.com/nodejs/node/issues/21825)). Direct **IDE `.exe`** spawns use **`windowsHide: false`** so Electron is not started with **`CREATE_NO_WINDOW`**. **Open in Explorer** no longer passes the target folder as the child process **`cwd`** (that could make **`explorer.exe`** fail asynchronously on some paths while the UI still reported success); the folder argument is **normalized** (backslashes, trailing `\` stripped except on drive roots).

- **NPU Text Tools:** **Core fix:** helper no longer sends **Ctrl+C/Ctrl+V** after the focus poll times out while Raycast is still foreground (that always copied nothing). It now **returns exit 2** with a clear stderr line, tries **AttachThreadInput + SetForegroundWindow** on the **z-order window below** Raycast, then re-checks. **`npm run test`** / **`npm run test:dotnet`** added. **Paste** success uses a **new** toast (hidden Phi toast was mutating wrong UI).
- **NPU Text Tools (selection):** After **`closeMainWindow`**, capture uses **`getSelectedText()`** first (Raycast API — avoids SendKeys/Ctrl+C when supported on Windows). Broadened **`RaycastDetector`** (paths / lowercase process names) for helper focus polling.

- **Phi bridges (`npu-notes-ext`, `npu-dev-toolbox-ext`, `npu-awake-ext`):** Added missing `TryUnlockNpuFeature()` call before any `LanguageModel` access. All three bridges were skipping the LAF unlock step, causing "Access is denied. Limited Access Feature is not available: com.microsoft.windows.ai.languagemodel" at runtime. `npu-text-tools-ext` was the only bridge with the unlock already in place. Token formula: `Base64(SHA256_UTF8("featureId!lafKey!pfn")[0..15])` — UTF-8 encoding is required (UTF-16LE produces Status 0).
